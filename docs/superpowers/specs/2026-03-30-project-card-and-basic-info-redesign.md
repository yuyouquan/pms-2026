# 项目卡片内容与基本信息重构设计文档

## 概述

基于2026年3月30日需求核对会议，对PMS系统进行以下核心改动：

1. **项目卡片内容按类型区分** - 不同项目类型（软件产品项目、整机产品项目、技术项目）显示不同字段
2. **项目状态映射** - 从IPM获取状态并进行映射处理
3. **基本信息数据扩展** - 项目详情中展示完整的基本信息字段（数据来源IPM/SCM）
4. **计划信息展示** - 计划开始时间、计划结束时间、开发周期等
5. **配置信息完善** - 系统权限与项目空间权限配置
6. **工作台优化** - 项目列表视图与待办中心改进

## 一、项目卡片内容

### 1.1 软件产品项目卡片

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 项目分类 | 取IPM | 固定为"软件产品项目" |
| 项目名称 | 取IPM | 如：IOSXXX |
| 项目状态 | 取IPM | 待立项/进行中/已完成/维护/暂停/已取消（需映射处理） |
| 计划开始时间 | 取IPM | 计划开始日期 |
| 计划结束时间 | 取IPM | 计划结束日期 |
| 项目经理 | 取IPM | SPM |

### 1.2 整机产品项目卡片

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 项目分类 | 取IPM | 固定为"整机产品项目"（需特殊处理） |
| 项目名称 | 取IPM | 如：X6877-D8400_H991 |
| 市场名 | 取IPM | 如：NOTE 50 Pro |
| 品牌 | 取IPM | 品牌信息 |
| 产品线 | 取IPM | 如：NOTE |
| 项目状态 | 取IPM | 待立项/进行中/已上市/维护/暂停/已取消（需映射处理） |
| 开发模式 | 平台自定义 | 如：ODC |
| 计划开始时间 | 取IPM | 计划开始日期 |
| 计划结束时间 | 取IPM | 计划结束日期 |
| 项目经理 | 取IPM | SPM |

### 1.3 技术项目卡片

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 项目分类 | 取IPM | 固定为"技术项目" |
| 项目名称 | 取IPM | 项目名称 |
| 项目状态 | 取IPM | 待立议/进行中/已完成/待验/已取消（需映射处理） |
| 计划开始时间 | 取IPM | 计划开始日期 |
| 计划结束时间 | 取IPM | 计划结束日期 |
| 项目经理 | 取IPM | SPM |

### 1.4 项目状态映射规则

IPM中的原始状态需要映射到PMS系统的展示状态：

- 筹备中 → 待立项
- 进行中 → 进行中
- 已完成 → 已完成
- 已取消 → 已取消
- 维护期 → 维护

> 注意：不同项目类型的状态枚举有差异，整机产品项目多了"已上市"状态。

### 1.5 卡片展示规则

- 所有类型的卡片大小一致
- 不同类型根据关键信息区分展示内容
- 卡片需展示进度条（百分比）
- 卡片底部展示SPM头像和最近更新时间

### 2.2 数据来源说明

- **IPM**：项目管理平台，提供项目基础信息
- **SCM**：供应链管理系统，提供产品规格信息
- **平台自定义**：PMS系统自有字段，用户手动编辑

### 2.3 基本信息展示分区

项目空间中的基本信息分为以下区块展示：

1. **项目基础信息** - 所有项目类型都显示
2. **市场项目信息** - 仅"整机产品项目"类型显示，按市场（OP/TR/RU等）分类展示

## 三、计划信息

### 3.1 计划信息字段

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 计划开始时间 | 取IPM | IPM中的计划开始时间 |
| 计划结束时间 | 取IPM | IPM中的计划结束时间 |
| 开发周期 | 计算字段 | 从里程碑中计算，去掉节假日和周末后的工作日天数 |
| 里程碑计划 | 贴张图展示 | 一级计划的横版视图甘特图缩略图 |

### 3.2 开发周期计算规则

- 开发周期 = 概念启动 到 最后一个STR阶段正式结束的时间
- 计划完成时间保持一致
- 需要去掉中间的节假日和周末
- 概念启动 = 项目正式阶段规划结束后立即启动

### 3.3 计划时间展示

- 计划完成时间显示在项目卡片和详情页
- 实际完成时间放在实际时间区域
- 计划的开发周期 与 实际的开发周期 分两列展示

## 五、权限配置

### 5.1 权限层级

系统权限分为两个层级：

1. **系统权限（配置中心权限）** - 全局配置权限
2. **项目空间权限** - 项目级别的权限控制

### 5.2 管理组

- 管理组分为三个组
- 管理组可编辑
- 管理者角色：管理者可以查看所有项目，可以编辑，可以导出

### 5.3 权限规则

- 项目路标：谁可以查看？谁可以编辑？谁可以导出？谁可以基线？
- 项目路标支持打基线功能
- 基线 = 在某个时间节点查看有多少项目，每个项目的状态是怎样的
- 打一个基线、时间基线，然后打一个快照tag

### 5.4 项目组成员权限

- 数据源最后一项为项目组成员
- 项目组成员的数据信息在基本信息中展示
- 不在权限里面的人，数据信息不展示给他（相当于不参与该项目）
- 跟NPM没关系，跟IPM没关系

## 六、工作台优化

### 6.1 工作台布局

工作台分为两个Tab模块：

1. **项目列表** - 展示项目卡片列表
2. **待办中心** - 展示用户的待办事项

### 6.2 项目列表

- 支持卡片视图和列表视图
- 列表视图使用表格展示，支持通用字段
- 支持全局搜索
- 支持项目状态筛选

### 6.3 待办中心

- 待办事项按分类展示
- 分类包括：待办、已完成、逾期等状态
- 逾期的任务需要告警高亮（红色高亮）
- 提醒类的任务用黄色高亮
- 支持飞书通知：
  - 告警通知：逾期的任务每天发飞书通知
    - 即将到期提醒：距到期前三天开始发通知
      - 新增任务通知：接收到新任务时发飞书通知
        - 任务修改通知：任务被修改时发通知给当事人

        ### 6.4 待办事项来源

        - 修订版本的计划中包含该用户，则生成待办
        - 一级计划创建修订版时生成待办
        - 计划阶段任务到期/逾期时生成告警

                                                                                  ## 八、技术决策

                                                                                  1. 复用现有的项目卡片布局，通过条件渲染不同字段
                                                                                  2. 基本信息使用分区Tab展示，市场信息按市场名分组
                                                                                  3. 状态映射通过配置表实现，便于后续扩展
                                                                                  4. Mock数据预留API接口替换能力
                                                                                  5. 待办中心的飞书通知为接口预留，不实际发送
                                                                                  6. 健康状态字段为新增维度，后续可扩展为自动计算

                                                                                  ## 九、注意事项

                                                                                  1. 不同项目类型的状态枚举不同，需要在前端做映射处理
                                                                                  2. 基本信息中部分字段来源于IPM，部分来源于SCM，需要标记清楚
                                                                                  3. 开发模式字段为平台自定义，不从外部系统获取
                                                                                  4. 开发周期的计算需要去掉节假日和周末
                                                                                  5. 权限过滤需要在前端实现：不在权限列表中的用户不展示相关数据
                                                                                  6. 市场项目信息只在整机产品项目类型下显示
