# 二级计划管理重构设计文档

## 概述

重构二级计划管理模块，包含三个核心改动：
1. 配置中心模板管理移除"需求开发计划"和"在研版本火车计划"两个固定类型
2. 项目空间中这两个计划固定显示在前两位且不可删除，使用专用界面
3. 实现"需求开发计划"（IR/SR/融合版 + 表格/甘特图）和"在研版本火车计划"（CRUD表格）的完整界面

## 改动范围

### 1. 配置中心 - 二级计划模板管理
**文件**: `src/app/config/level2-template/page.tsx`
**改动**: `LEVEL2_PLAN_TYPES` 数组移除前两项，保留 `['1+N MR版本火车计划', '粉丝版本计划', '基础体验计划', 'WBS计划']`

### 2. 类型定义扩展
**文件**: `src/types/index.ts`
新增类型：
- `IRRequirement` - IR需求数据结构
- `SRRequirement` - SR需求数据结构
- `VersionTrainRecord` - 在研版本火车记录
- `VersionCategory` - 版本分类枚举
- `VersionTrainStatus` - 版本状态枚举

### 3. 需求开发计划组件
**新文件**: `src/components/plans/RequirementDevPlan.tsx`

**功能**:
- 三Tab切换: IR需求 / SR需求 / 融合版(待定)
- 表格/甘特图视图切换
- 全字段搜索
- 自定义列配置

**IR需求表格字段**: 序号, 领域, IR编号, IR标题, 优先级, IR状态, 需求测试计划开始时间, 需求测试计划完成时间, 需求验收计划开始时间, 需求验收计划完成时间

**IR甘特图**: 分需求测试维度和需求验收维度（通过子Tab切换）
- 需求测试: 序号, 领域, IR编号, IR标题, 优先级, IR状态, 计划开始时间, 计划完成时间, 甘特图
- 需求验收: 同上字段，时间取验收计划时间

**SR需求表格字段**: 序号, SR编号, SR标题, 关联IR, 开发部门, SR状态, 计划转测版本, 实际转测版本, 需求测试计划开始时间, 需求测试计划完成时间, 需求验收计划开始时间, 需求验收计划完成时间

**SR甘特图**: 同IR，分需求测试/需求验收维度

### 4. 在研版本火车计划组件
**新文件**: `src/components/plans/VersionTrainPlan.tsx`

**功能**:
- 表格展示，按版本号升序排列
- 多维度搜索、分页、自定义表头、数据导出
- CRUD操作（新建/编辑Modal + 删除确认）

**表格字段**:
| 字段 | 数据来源 |
|------|----------|
| 序号 | 自动排序 |
| 版本分类 | 编辑填写 |
| 状态 | 编辑填写 |
| 计划编译时间 | 编辑填写 |
| 计划转测时间 | 编辑填写 |
| 计划测试开始时间 | 编辑填写 |
| 计划测试完成时间 | 编辑填写 |
| 主测机型 | 编辑填写 |
| 版本目标 | 编辑填写 |
| 实际编译时间 | 系统联动(Mock) |
| 实际转测时间 | 系统联动(Mock) |
| 实际测试开始时间 | 系统联动(Mock) |
| 实际测试完成时间 | 系统联动(Mock) |
| 关联需求清单 | 点击弹窗展示 |
| 关联缺陷清单 | 点击弹窗展示 |
| 附件 | 上传/下载 |
| 操作 | 修改/删除 |

**新建Modal字段**: 版本号(文本), 版本分类(下拉:主干版本/量产版本), 主测机型(下拉), 状态(下拉:已完成/自检中/测试中/未开始,默认未开始), 版本目标(多行文本), 计划编译/转测/测试开始/完成时间(日期), 附件(多文件上传)

### 5. 项目空间集成
**文件**: `src/app/page.tsx`

**改动**:
- 二级计划Tab列表中，"需求开发计划"和"在研版本火车计划"始终排在最前，不可删除
- 点击这两个计划时渲染专用组件
- 创建二级计划的下拉选项中排除这两个类型
- Mock数据移至各组件内部

## 技术决策
- 复用现有 `DHTMLXGantt` 组件，扩展列配置能力
- 复用 `.pms-table` / `.pms-modal` 样式体系
- Mock数据内置在组件中，通过props接口预留API替换能力
- 附件上传使用antd `Upload`组件模拟，不实际上传
