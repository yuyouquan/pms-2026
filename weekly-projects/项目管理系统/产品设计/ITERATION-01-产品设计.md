## 1. 需求分析

### 1.1 需求背景

本项目为项目管理系统，旨在帮助企业实现项目计划的有效管理和执行监控。系统需要支持多种项目类型的全生命周期管理，从项目立项到计划制定、执行、跟踪的全流程管理。

### 1.2 需求概述

根据原始需求文档前5行，本迭代核心需求如下：

1. **系统架构**：系统分为工作台和配置中心两大模块
2. **配置管理**：计划管理功能，包含三级活动序号规则（一级活动1/X、二级活动1.1/X.X、三级活动1.1.1/X.X.X）
3. **项目类型区分**：计划管理需要按项目类型区分（整机产品项目、产品项目、技术项目、能力建设项目）
4. **计划层级**：每个项目类型需要区分一级计划和二级计划
5. **一级计划模板管理**：版本下拉切换、创建修订、编辑、发布、版本对比功能

### 1.3 核心功能清单

| 序号 | 功能模块 | 功能描述 | 优先级 |
|------|----------|----------|--------|
| 1 | 配置中心-一级计划模板管理 | 支持版本管理、创建修订、编辑、发布、版本对比 | P0 |
| 2 | 配置中心-二级计划模板管理 | 支持6种固定类型+自定义类型，版本管理 | P0 |

## 2. 功能设计

### 2.1 配置中心

#### 2.1.1 一级计划模板管理

**页面结构**：
- 顶部：项目类型下拉选择器、版本下拉选择器
- 操作栏：创建修订、编辑、发布、历史版本对比按钮
- 内容区：计划表格/甘特图

**版本规则**：
- 版本号格式：V1、V2、V3...（自增）
- 状态标识：已发布、修订版
- 修订版存在时，隐藏"创建修订"按钮
- 仅修订版显示"编辑"和"发布"按钮

**版本操作流程**：
```
1. 创建修订 → 生成新修订版（克隆已发布版本或空版本）
2. 编辑 → 进入修改模式，可拖拽排序、添加子项
3. 保存 → 保存修改到修订版
4. 发布 → 修订版变为已发布
5. 版本对比 → 选择两个版本对比差异（增/删/改）
```

**序号规则**：
- 一级活动：1, 2, 3...
- 二级活动：1.1, 1.2, 2.1...
- 最多创建二层活动

**表格字段**：
| 字段 | 可编辑 | 说明 |
|------|--------|------|
| 序号 | 否 | 自动生成 |
| 任务名称 | 是 | 单行文本 |
| 责任人 | 否 | - |
| 前置任务 | 是 | 关联其他任务 |
| 计划开始时间 | 否 | - |
| 计划完成时间 | 否 | - |
| 预估工期 | 否 | - |
| 实际开始时间 | 否 | - |
| 实际完成时间 | 否 | - |
| 实际工期 | 否 | - |
| 状态 | 否 | - |
| 操作 | - | 删除 |

**操作功能**：
- 拖拽排序：改变顺序和层级
- 添加子项：在序号下添加子活动
- 删除：二次确认后删除（含子任务一并删除）

#### 2.1.2 二级计划模板管理

**计划类型**：
- 固定类型（6种）：需求开发计划、在研版本火车计划、1+N MR版本火车计划、粉丝版本计划、基础体验计划、WBS计划
- 自定义类型：用户可添加

**序号规则**：
- 一级活动：1, 2, 3...
- 二级活动：1.1, 1.2, 2.1...
- 三级活动：1.1.1, 1.1.2...
- 最多创建三层活动

**版本管理与一级计划模板相同**

### 2.2 工作台

#### 2.2.1 项目卡片

**项目字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 项目名 | 文本 | 如：X6877-D8400_H991 |
| 项目类型 | 下拉 | 整机产品项目/产品项目/技术项目/能力建设项目 |
| 安卓版本 | 下拉 | Android 15/16/xxx |
| 芯片平台 | 下拉 | MTK/QCOM/UNISOC |
| 项目状态 | 下拉 | 筹备中/进行中/生命周期/上市 |
| SPM | 下拉 | 人员选择 |
| 市场 | 下拉 | OP/TR/RU（仅整机产品项目显示） |

**交互**：点击卡片跳转至项目空间

#### 2.2.2 项目空间

**十大模块**：
1. 基础信息
2. 概况（待定）
3. 需求（待定）
4. 计划
5. 资源（待定）
6. 任务（待定）
7. 风险（待定）
8. 缺陷（待定）
9. 团队（待定）
10. 项目文档（待定）

**基础信息模块**：
- 项目基础信息（所有项目）
- 市场项目信息（仅整机产品项目）

#### 2.2.3 计划模块

**一级计划管理**：
- 功能同配置中心一级计划模板
- 额外支持甘特图视图
- 初始创建修订时，从配置中心模板回填数据

**二级计划管理**：
- 支持创建多种类型二级计划
- 需先有已发布的一级计划才能创建
- 绑定里程碑（从一级计划二级活动选择）
- 除"需求开发计划"外，其他类型功能同配置中心

**需求开发计划**：
- 展示IR需求、SR需求、融合版
- 表格/甘特图视图切换

**计划总览**：
- 融合一级计划和二级计划
- 按时间自动拆分和挂靠

### 3.2 关键交互设计

**版本下拉切换**：
- 显示格式：V1(已发布)、V2(修订版)
- 点击切换显示不同版本详情
- 初始化显示最新版本

**创建修订流程**：
1. 检查是否存在修订版（存在则隐藏按钮）
2. 点击创建 → 生成新修订版（克隆或空）
3. 自动跳转至新修订版

**编辑模式**：
- 顶部显示版本号 + 保存/取消按钮
- 所有字段可编辑
- 支持拖拽排序
- 支持添加子项

**版本对比**：
- 弹出框选择两个版本
- 展示差异：绿色(增)、红色(删)、蓝色(改)

### 4.2 序号生成规则

```typescript
function generateTaskNumber(tasks: PlanTask[], parentId?: string): string {
  // 获取同级任务
  const siblings = parentId 
    ? tasks.find(t => t.id === parentId)?.children || []
    : tasks.filter(t => !t.parentId);
  
  if (!parentId) {
    // 一级活动：1, 2, 3...
    return String(siblings.length + 1);
  } else {
    // 子活动：1.1, 1.2, 2.1...
    const parent = tasks.find(t => t.id === parentId);
    const parentNumber = parent?.id.split('.').pop() || '1';
    return `${parentNumber}.${siblings.length + 1}`;
  }
}
```
